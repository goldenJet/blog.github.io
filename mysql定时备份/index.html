mysql定时备份

Posted on 2017-11-23 | In Linux

此文介绍了比较初级但是易上手并且较简介的定时备份mysql数据库的方法。

一、说在前面

        本文介绍的方法是利用 mysql 的 mysqldump 的命令来备份mysql数据库，

        步骤是编写一个shell 脚本，脚本内容就是备份数据库的命令，然后我们再定时去执行这个脚本就达到了定时备份数据库的效果。

二、新建 shell 脚本

        需要注意的是，如果是在 windows 环境下编辑脚本内容，切记：文件格式要转换成 Unix 格式，否则 sh 脚本运行会出错。

        附上脚本内容，内有注释：

        vim /data/mysql/mysqlBackup.sh

三、权限

        修改 shell 脚本属性，赋予执行权限

        chmod 600 /data/mysql/mysqlBackup.sh

        chmod +x /data/mysql/mysqlBackup.sh


四、定时执行（crontab）

        利用crond 来周期性执行我们的备份任务。

        但是前提是，必须安装该服务。

        有两种方式来进行，分系统任务调度和用户任务调度两种，第一种是修改全局的 /etc/crontab 配置文件，第二种是命令行进行修改用户的任务。

        ① 安装：

            yum -y install vixie-cron

            yum -y install crontabs

            第一个是 cron 的主程序，第二个是用来安装、卸装、或列举用来驱动 cron 守护进程的表格的程序。

        ② 配置服务：

            查看服务状态：service crond status

            启动服务：       service crond start

            关闭服务：       service crond stop

            重启服务：       service crond restart

            重新载入配置：service crond reload

            在CentOS系统中加入开机自动启动: chkconfig –level 345 crond on

        ③ 配置自动执行文件

            cron 默认的自动执行文件是：/etc/crontab

            vim /etc/crontab

            

            如图所示，

            前四行是用来配置 cron 任务运行环境的变量。
            SHELL 变量的值告诉系统要使用哪个 shell 环境（在这个例子里是 bash shell）；
            PATH 变量定义用来执行命令的路径。
            cron 任务的输出被邮寄给 MAILTO 变量定义的用户名。
            如果 MAILTO 变量被定义为空白字符串（MAILTO=””），电子邮件就不会被寄出。
            HOME 变量可以用来设置在执行命令或脚本时使用的主目录。     

            中间部分是表达式简述。

            最后两行是我们自己添加进去的定时任务。             

            00 /5 /data/mysql/test_mysqlBackup.sh 

                每隔5个小时执行一次脚本

            00 03 * /data/mysql/prod_mysqlBackup.sh

                每天的凌晨3点执行一次脚本

         ④ 第二种方式设置定时任务（命令方式）

                语法： crontab[-u <用户名称>][配置文件] 或 crontab [-u <用户名称>][-elr]

                参数：
                    -e　编辑该用户的计时器设置。
                    -l　列出该用户的计时器设置。
                    -r　删除该用户的计时器设置。
                    -u<用户名称> 　指定要设定计时器的用户名称。

                举例：

                    列出当前用户的定时任务：crontab -l

                    列出root用户的定时任务：crontab -uroot -l

                    编辑当前用户的定时任务：crontab -e

            ⑤ 查看日志

                查看 crontab 执行记录：tail -f /var/log/cron

                但是上述文件只记录执行与否，具体的执行情况需要查看 linux 给 用户发送的邮件

                    如，root 用户：tail -f /var/spool/mail/root

              ⑥ 错误排查

                     如果有的时候碰到脚本并没有执行，那么就先看执行记录：tail -f /var/log/cron

                     如果有记录，在查看邮件内的错误信息。

                     其实很多情况下的错误原因是由于没有在 root 的环境下执行罢了，并且，脚本内应该使用绝对路径。

                     所以，我们在用户的任务调度下修改如下：crontab -e

                            

                    另外，执行脚本内，也最好改为绝对路径。


五、还原数据库

        ① 由于我们备份的数据库是 .gz 文件，所以我们需要先解压缩：

            使用 【gunzip 文件】 命令解压，如：gunzip testbak-201711181257.sql.gz

            解压之后便得到一个 sql 执行文件，然后执行这个 sql 文件便达到了恢复的效果。

        ② 恢复

            使用【mysql -u username -p databse < backup.sql】命令来恢复，如：mysql -u root -p testbak < testbak-201711181257.sql
